Geo-Bullet

Simulation of granular media by Bullet physics engine with Geotechnical engineering perspective.

Bullet physics engine

Bullet is a physics engine initially developed for video games and animations purposes. However, it is written based on physical laws. It is fast,robust and handy. To gain more information on Bullet, readers can view Bullet on GitHub. Bullet consists of three major modules:

  • Rigid body module: which is based on contact dynamics approach.
  • Soft body module: which is based on spring-mass systems (application of FEM is also feasible).
  • Fluid module: using smoothed particle hydrodynamics (SPH)- coupling is in progress...

Besides, Bullet comes with very nice features, such as Voronoi fracturing, CAD file importers, etc.

Simulation of granular media using Bullet

Since Bullet uses contact dynamics as rigid body solver, soil granules can be modelled as discrete objects with a wide range of geometries. In addition, in the cases in which deformable bodies such as reinforcements, membranes, etc. are associated in the simulation, they can be modelled as softbodies.

Usage

In order to use GeoBullet, Bullet must be installed already (preferably in ~/Bullet/bullet3/) folder. Moreover, 'Makefile', 'python' and 'Matplotlib' are recommended to be installed. Every folder in the code repository represent an example, and the codes are written in a way to be self-explanatory, so, with less effort it can be understood. The file 'variables.h' in each folder represents the main variables and the associated magnitudes for the simulations. For visualisation purposes, several writers are provided. The output of the writers are recommended to be opened by paraview software.
